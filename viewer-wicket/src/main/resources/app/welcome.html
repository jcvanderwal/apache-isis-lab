This sample webapp was generated by the Isis QuickStart archetype, configured to use Isis' wicket viewer.


By default the domain objects are persisted using JDO/DataNucleus objectstore, configured to persist to an in-memory HSQL/DB database.


To change the JDBC connection string, edit WEB-INF/persistor_datanucleus.properties. Also update the pom.xml for the appropriate JDBC driver.


To change the objectstore implementation, edit the WEB-INF/isis.properties file.


The wicket viewer initially is configured using Wicket's 'development' mode, with Isis' authorization modules are disabled. For deployment, typically Wicket viewer should be switched to 'deployment' mode, by editing WEB-INF/web.xml. Isis' authorization should also be enabled (again, by editing the WEB-INF/isis.properties config file).
